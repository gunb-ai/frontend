Contract drift from one declaration

Real source: dsl/examples/weather/weather.dag. Add a variant to Condition; every consumer either updates or the build fails closed.

module examples.weather

type Condition
  = Sunny
  | Cloudy
  | Rainy { mm_per_hour: Float }
  | Snowy { cm_per_hour: Float }

fn describe_condition(cond: Condition) -> String {
  match cond {
    Sunny => "clear skies"
    Cloudy => "overcast"
    Rainy { mm_per_hour: rate } =>
      if rate > 10.0 { "heavy rain" } else { "light rain" }
    Snowy { cm_per_hour: _ } => "snowfall"
  }
}

Composed cost drift across the graph

Roadmap lens story: a local change (e.g. index lookup → linear scan) composes through callers and trips a CI receipt before merge.

Built for agents: edit the graph, not the text

Most coding agents operate on raw bytes — write a string, hope it parses. daglang exposes the program as a typed graph: agents read a scope, run a lens, apply a structural diff, and re-emit source only after the graph checks pass. The flow below mirrors the v4 architecture where CoreNode is the canonical input and apply_diff is symmetric to fold_node.

read graph → run lens → apply diff → check → emit

Programmatic surface (roadmap sketch)

Same primitives, available to tools and agents directly. Reads are folds; writes are typed diffs; emission is the final step after the graph is green.

gunbc · TypeScript SDKBuilding next
import { gunbc, replaceNode } from "@gunb-ai/gunbc";

const graph = await gunbc.read("examples/weather.dag");

// 1. Inspect & analyze
const fn = graph.fn("freezing_locations");
const report = await graph.lens("complexity", { scope: fn });
// report.class === "O(n)"

// 2. Propose a structural diff
const candidate = graph.applyDiff([
  replaceNode(fn.body.find, {
    kind: "index_lookup",
    index: "customers_by_id",
  }),
]);

// 3. Validate before emitting
const check = await candidate.check();
if (!check.ok) throw new Error(check.diagnostics.join("\n"));

// 4. Emit target source from the green graph
await candidate.emit({ target: "rust", outDir: "/tmp/weather-out" });

Reads are fold_node; writes are apply_diff. Same canonical CoreNode whether the input came from text, an agent, or a round-trip translation — so an agent's patch is held to the same lenses as a human edit.

Beyond programming languages

A target doesn't have to be a language. If the artifact has structure — a circuit, a schema, a protocol — it can be modeled and kept in lockstep with the rest of the graph. Both examples below are Vision tier; the .dag shapes are illustrative.

module Counter {
  input  clk:   Clock
  input  reset: Bool
  state  count: UInt<8>

  on rising(clk) {
    if reset { count = 0 }
    else     { count = count + 1 }
  }
}

module Counter (

  input         reset,
