Contract drift from one declaration

Real source: dsl/examples/weather/weather.dag. Verified gate (Rust): compile with gunbc compile --target rust then cargo check on emitted output.

.dag sourcev0.1.0
module examples.weather

type Condition
  = Sunny
  | Cloudy
  | Rainy { mm_per_hour: Float }
  | Snowy { cm_per_hour: Float }

fn describe_condition(cond: Condition) -> String {
  match cond {
    Sunny => "clear skies"
    Cloudy => "overcast"
    Rainy { mm_per_hour: rate } =>
      if rate > 10.0 { "heavy rain" } else { "light rain" }
    Snowy { cm_per_hour: _ } => "snowfall"
  }
}

Complexity analyzer — current output

This is the current v2 in-compiler complexity analyzer. The broader lens-gated CI receipt path is roadmap. The compiler runs complexity.dag during compile and attaches function_classes to analyzed functions. Output below is from real daglang sources, not mocked.

Current: per-function cost classes

dsl/examples/weather/weather.dag v0.1.0 · real .dag 
fn freezing_locations(forecasts: List<Forecast>) -> List<String> {
  forecasts
    |> filter(f => is_freezing(temp: f.low))
    |> map(f => f.location)
}

fn daily_summary(forecast: Forecast) -> String {
  let high_f = to_fahrenheit(temp: forecast.high)
  let desc = describe_condition(cond: forecast.condition)
  concat(forecast.location, ": ", desc, ", high ",
         to_string(value: high_f), "°F")
}

Preview: composed cost drift across the dependency graph

Roadmap lens story: a local change (e.g. lookup from index to linear scan) composes through callers. Not a shipped CI receipt in v0.1.0.

.dag (illustrative)preview
fn processOrders(orders: List<Order>) -> List<Receipt> {
  orders |> map(o => enrich(o))
}

fn enrich(order: Order) -> Receipt {
  let customer = lookupCustomer(id: order.customer_id)
  ...
}

fn lookupCustomer(id: UserId) -> Customer {
  customers |> find(c => c.id == id)   // was O(1) index; now linear scan
}

suggested shape: index customers by id before enrich

Parser-style SCC descent is analyzed today (e.g. parse_items → O(n)). Nested folds on the same collection may classify as O(n) when identity is tracked. See complexity.dag.
