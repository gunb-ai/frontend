Every feature, on one circuit.

One protagonist — a low-pass RC filter — walked through every daglang feature so a developer who's never touched an oscilloscope can see why "code as graph" matters. The sliders above drive the graph. Every artifact on this page is a projection of that same graph and updates in lockstep.

Step 1 · the graph

A 27-line .dag file. That's it.

The developer writes typed components and how they connect — same vocabulary the v4 SPICE substrate uses internally. No SPICE syntax. No graphics code. The graph is the single source of truth.

low_pass.dag your code · single source of truth 
module examples.low_pass

import v4.extdeps.formats.spice {
  Component, DeckStatement, Directive,
  NodeRef, SpiceNetlist, SpiceValue
}

data n_in:  NodeRef = Named { name: "in"  }
data n_out: NodeRef = Named { name: "out" }
data gnd:   NodeRef = Ground

circuit LowPassFilter {
  spec  cutoff_band: (800 Hz, 3.2 kHz)

  vin: VoltageSource { ac: 1.0V,           // drives "in"
                       n_plus: n_in, n_minus: gnd }
  r1:  Resistor      { value: 1.0 kΩ,
                       n_plus: n_in, n_minus: n_out }
  c1:  Capacitor     { value: 100 nF,
                       n_plus: n_out, n_minus: gnd }
  output vout = n_out
}

Step 3 · affected_set

Touch a value. Watch which artifacts must re-emit.

The amber sweep across a tile means the graph's affected_set primitive returned that artifact's emit-path for the current edit. Move the R slider — five of six tiles light up. Notice that generated docs stays cold: the topology didn't change, only a parameter value, so the Markdown projection is structurally unaffected. That's the difference between a build-graph and a glob.

Real affected_set(root, { at }) walks the typed CoreNode graph and returns the closure of dependent nodes. CI re-runs only the lenses whose scope intersects this set; cached results carry their content-hash receipt forward for everything else.

Step 4 · lens gate

Structural facts checked before any artifact ships.

The circuit declared a cutoff spec: 800 Hz < f_c < 3.2 kHz. A spec-band lens folds across the graph, computes f_c symbolically (not from a simulator run), and either passes or fails closed. Drive R below ~250 Ω or above ~6 kΩ — the lens turns red. Nothing emits when it's red.

Step 5 · the agent path

An agent makes the same edit. Same gates. Same receipt.

The slider was a UI affordance. Underneath, it's a typed Diff against the CoreNode. An agent can call exactly the same primitives — no special path, no "agent mode" — and the same lens either gates the edit or admits it. Reads are fold_node; writes are apply_diff; emission is the last step.

agent.ts — same path as the slider above Building next · v4 SDK 
import {
  ingestText, foldNode, affectedSet,
  applyDiff, emit, replaceNode,
  type CoreNode, type Diff, type Outcome,
} from "@gunb-ai/gunbc";

const root: CoreNode = await ingestText(
  await Deno.readTextFile("examples/low_pass.dag"),
  { inputLang: "dag" },
);

// 1 · Diff: same shape as moving the R slider in the UI
const diff: Diff = {
  edits: [
    replaceNode("circuit:LowPassFilter/r1/value",
                SpiceValue({ lexeme: "470" })),  // was "1k"
  ],
};

// 2 · Candidate: apply_diff sequentially folds substitutions through.
//     Any unresolved Path = whole Diff fails closed (P3).
const candidate: Outcome<CoreNode> = applyDiff(root, diff);
if (!candidate.ok) throw candidate.diagnostics;

// 3 · Affected: surgical re-check, not whole-program rebuild
const affected = affectedSet(candidate.value, {
  at: "circuit:LowPassFilter/r1/value",
});
//   → ["spice:netlist", "ts:FilterEditor", "openapi:FilterParams",
//      "rust:simulate", "bode:H(f)"]   // docs not in set · topology unchanged

// 4 · Gate: spec_band_lens folds across the graph, compares to declared band
const gated = foldNode(candidate.value, specBandLens);
//   → { spec_band: "FAIL · f_c = 3.39 kHz · outside 800–3200 Hz" }
//   ⇒ emit() refuses · no artifact files written

The agent never wrote a string. It walked the typed graph, proposed a structural Diff, and the lens caught the spec violation before any SPICE deck or TS file changed. Same surface a future query-driven rewrite or self-modification path uses.

Step 6 · omni-ingestion

Reality changes the SPICE deck. The graph absorbs it.

An EE on the team hand-edits the netlist on the bench — replaces R1 with a 470 Ω resistor they had on hand. That edit shouldn't strand the rest of the system. Omni-ingestion runs the SPICE grammar in reverse, lifts the deck back to a SpiceNetlist value, derives the resulting CoreNode diff, and propagates it — the UI knob settles to 470 Ω, the Bode plot moves, and the same lens runs.

ingest_from_spice.ts — the inverse arrow of emit Building next · round-trip translate 
import { ingestText, foldNode, diffAgainst, emit }
  from "@gunb-ai/gunbc";

// Bench engineer hand-edited the deck:
//   R1 in out 470     ← was 1k
const handEdited = await Deno.readTextFile("/bench/low_pass.cir");

// 1 · Parse SPICE → SpiceNetlist (Shape B inverse) → lift to CoreNode.
//     Same shim machinery as ingestText — different input grammar.
const lifted = await ingestText(handEdited, { inputLang: "spice" });

// 2 · Compute the structural diff against the canonical graph.
const original = await ingestText(
  await Deno.readTextFile("examples/low_pass.dag"),
  { inputLang: "dag" },
);
const diff = diffAgainst(original, lifted);
//   → 1 edit · path = "circuit:LowPassFilter/r1/value" · 1k → 470

// 3 · Re-emit the sibling artifacts from the lifted graph.
//     The TS UI redraws with the slider at 470Ω; the Bode plot moves;
//     the lens runs over the same band. No artifact diverges.
await emit(lifted, { target: "typescript", outDir: "./ui/src" });
await emit(lifted, { target: "rust",       outDir: "./svc/src" });

Same primitives, opposite direction. omni-emission projects the graph outward; omni-ingestion pulls reality back in. Together they make every artifact a peer of the source — no canonical-format tyranny, no manual reconciliation.
