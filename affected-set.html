Graph-aware impact analysis

Change one line. See the graph move.

A source line is just a view. gunbc resolves it to typed graph nodes — fields, variants, call edges, match arms, literals, effect boundaries, target-artifact projections. When a node changes, the compiler walks the graph to every downstream node whose meaning could shift. Generated checks and lenses then run only on that affected scope — across functions, modules, and emitted languages. (In the compiler architecture, this is the affected_set primitive over a typed CoreNode.)

Not file-by-file. Not function-by-function. Node-by-node.

Compute is local: the scenarios are precomputed for clarity, but real affected_set(root, { at }) in v4 returns the dependency closure of the edit Path against the typed graph — not a textual grep, and not a whole-program rebuild. Combined with sequential apply_diff and candidate-state gating, an agent's structural patch either passes the same lenses a human edit would, or fails closed before any file on disk changes.

Not function-by-function. Node-by-node.

A function isn't the smallest unit gunbc can reason about. A field, a match arm, a variant, a call edge, even a literal — each is a graph node with its own dependents, generated checks, and target-language projections. Line-level UX, node-level truth. The interface lets a human start from a line; the lens reasons about the nodes that line resolves to.

An agent edits the graph through the same pipeline.

Same primitives, no special "agent mode." An agent's structural Diff is held to the same lenses and gates a human edit triggers, and re-emits only the affected artifacts.

read graph → locate node → apply_diff → affected_set → testgen + lenses → emit affected only 
// roadmap API shape — composes the primitives from
// design-v4-compiler-homomorphism.md
const graph = await gunbc.read("examples/weather.dag");

// 1 · Locate: the user (or agent) starts from a line; gunbc resolves it
//     to graph nodes — the addressable unit, not the line.
const edit = graph.editAtLine(7).addVariant(
  "Hail", { mm_per_hour: "Float" },
);

// 2 · Diff: apply_diff substitutes through a candidate root; fails closed
//     if any Path in the Diff can't resolve in the current state (P3).
const candidate = await graph.applyDiff(edit);

// 3 · Affected: typed dependency closure from the changed Path
const affected = await candidate.affectedSet({
  at: "Type.Condition",
});
//   → ["Match.Condition", "Call.describe_condition",
//      "Rust::weather.rs::Condition", "TS::weather.ts::Condition",
//      "Python::weather.py::Condition", "OpenAPI::WeatherCondition"]

// 4 · Generated checks for ONLY the affected scope
const checks = await candidate.generatedChecks({ scope: affected });
// → exhaustiveness fails: describe_condition missing arm: Hail

// 5 · Re-emit only the affected artifacts; skipped ones carry receipts
await candidate.emitAffected({ scope: affected });
// emitted: weather.rs · weather.ts · weather.py · WeatherCondition.yaml
// skipped: PSpice low_pass · OpenAPI auth · 27 unrelated
