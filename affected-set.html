affected_set · v4 substrate primitive

Change one line. See every downstream node light up.

The compiler's affected_set primitive walks the typed CoreNode graph from an edit site outward, returning every node whose meaning could shift because of that edit. This is what powers incremental rebuild, lens re-runs, and agent-native apply_diff: only the affected scope has to be re-checked, and a typed diff that breaks anything downstream fails closed before it can re-emit text. Compute is local: the scenarios are precomputed for clarity, but real affected_set(root, { at }) in v4 returns the dependency closure of the edit Path against the typed graph — not a textual grep, and not a whole-program rebuild. Combined with sequential apply_diff and candidate-state gating, an agent's structural patch either passes the same lenses a human edit would, or fails closed before any file on disk changes.
