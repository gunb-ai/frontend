primitives · 01

Materials

Five materials. Each has a reason. Moss is structural continuity, not "success." Clay is a boundary, not "danger." Neither is a UI status color — no traffic-light usage anywhere in the system.

Voidmaterial

The ground. Near-black, slightly warm. Product surface.

Stonematerial

Warm grey. Primary text on dark. Patient, recessive.

Warm Whitematerial

Off-white, warm cast. Light-mode surface.

Mossaccent

Structural continuity. Link, held relation. Not "success."

Clayaccent

Boundary. Diagnostic, invariant break. Not "danger."

tutorial · 14

From primitives to value

Three movements. Atoms compose into a structure (1). The compiler reads that structure into target artifacts, stopping where the reach ends (2). Independent lenses each read the same structure to extract different value — without knowing about each other (3). Each section is candidate copy for the public site, used in sequence to build the argument without ever showing a feature grid.

movement 1 · the structure

A system is nodes and edges

Before anything else, the system has to exist as structure. Atoms compose: a node is a unit; an edge is a relation; together they form a graph. Same atoms describe a function signature, a service mesh, or a circuit. The compiler doesn't care which.

01Node

A unit of structure. A type, a function, a service — anything addressable.

vertex

02Edge

A relation. One node depends on, references, or projects into another.

directed arc

03Graph

Atoms compose into the system's description. Typed, directed, addressable end to end.

typed dag

Once the system is structure, the compiler can read it.

movement 2 · reading the structure

The compiler reads structurally and emits

Reading is structural. Each node's meaning depends only on its children — a fold. The same fold can project to any target the compiler has a translation for: Rust, TypeScript, OpenAPI, tests, docs. Where the structure stops reaching, the compiler stops emitting. Successful reads leave receipts.

04Fold

Each node read in terms of its children. Local rules combine into a global meaning.

catamorphism

05Projection

From structure to a target artifact. The same fold can project into Rust, schema, tests, docs.

structure-preserving map

06Boundary

Where structure ends. Beyond is unmodeled — no artifact emitted, no plausible output invented.

fails closed

07Checkpoint

A held invariant. The receipt of a successful read — what's true, where, and at what cost.

witness

Reading once gives one artifact. The same reading, restricted to a subset, becomes a measurement.

movement 3 · what falls out

Independent lenses on the same structure

The same reading machinery, restricted in different ways, becomes different value. Affected-set asks "which subset depends on this change?" Complexity asks "what's the cost class over this subset?" Testgen asks "what cases must I cover to exhaust this subset?" Omni-emission asks "what targets can this subset project into?" Each lens reads the same structure. None know about the others. They're orthogonal — and they compose without interfering.

08Affected-set

Change one node. The compiler computes exactly which nodes downstream depend on it — and only those.

dependency closure

09Complexity

Read the structure, report the cost class. The same fold that emits an artifact can measure its growth.

static cost over the fold

10Testgen

Read the structure, enumerate the cases. Every variant of every node becomes a test that must pass.

exhaustive enumeration

11Omni-emission

Same structure, many target artifacts. Backend types, client types, schemas, docs — each derived, each consistent because each comes from the same source.

multiple projections

Four lenses, one structure. Each is independent of the others; together they're the compounding payoff for describing the system once.

gunbc · structural compiler

A structural compiler for composable systems.

Describe the system once. Let the compiler derive the rest.

Modern systems are full of hand-written translations: backend types to client types, models to schemas, services to docs, code to tests, source to build rules.

daglang describes the system once. gunbc checks that description and derives the mechanical pieces. When the description is incomplete, the compiler refuses to guess.
