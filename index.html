A compiler for systems that can’t drift.

Declare your program as a typed graph. gunbc runs structural lenses over that graph, then emits target artifacts from the same source — one graph, many artifacts, structural checks before code ships.

What v0.1.0 actually does

Correctness claims apply to the supported subset only. Unsupported paths fail closed with named diagnostics — not plausible fake output.

Catch cost drift before runtime

Lenses are structural queries over the graph, not grep or late scans. A complexity lens reports composed cost of a change before you ship. Concept demo — preview.

Generate targets from one source

Emit Rust, Go, and Python from the same .dag declarations. Change a field once — every derived artifact updates or the build fails.

Fail closed, don’t fake it

Unsupported constructs return named diagnostics at the boundary. No silent “best effort” emission on paths outside the verified surface.

Lead example — contract drift

From dsl/examples/weather/weather.dag in the daglang repo — types, enums, and pure functions that compile to Rust today.

User-facing types stay in one declaration v0.1.0 · Rust verified 
type Forecast {
  location: String
  high: Temperature
  low: Temperature
  condition: Condition
}

type Condition
  = Sunny | Cloudy
  | Rainy { mm_per_hour: Float }
  | Snowy { cm_per_hour: Float }
→ Rust enum + struct emission
→ Go / Python on supported v0.1.0 surface
→ Add Refunded to Condition without updating consumers → build fails closed
